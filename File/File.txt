<div class="heading-wrapper" data-heading-level="h2"><a class="anchor-link docon docon-link" href="#remarks" aria-label="Раздел с заголовком: Замечания"></a><h2 id="remarks" class="heading-anchor">Замечания</h2></div>
<p>Если ошибка, <strong><code>new</code></strong> возвращается ноль или вызывает исключение. Дополнительные сведения см. в разделе <a href="https://learn.microsoft.com/ru-ru/cpp/cpp/new-and-delete-operators?view=msvc-170" data-linktype="relative-path">"Операторы" и <code>delete</code> "<code>new</code>Операторы</a>". Это поведение по умолчанию можно изменить, написав настраиваемую подпрограмму обработки исключений и вызвав <a href="https://learn.microsoft.com/ru-ru/cpp/c-runtime-library/reference/set-new-handler?view=msvc-170" data-linktype="relative-path"><code>_set_new_handler</code></a> функцию библиотеки времени выполнения с именем функции в качестве аргумента.</p>
<p>Сведения о создании объекта в управляемой куче в C++/CLI и C++/CX см <a href="https://learn.microsoft.com/ru-ru/cpp/extensions/ref-new-gcnew-cpp-component-extensions?view=msvc-170" data-linktype="relative-path">. в разделе gcnew</a>.</p>
<div class="alert is-info">
<p class="alert-title"><span class="docon docon-status-error-outline" aria-hidden="true"></span> Примечание</p>
<p>Расширения компонентов Microsoft C++ (C++/CX) поддерживают ключевое <strong><code>new</code></strong> слово для добавления записей слотов vtable. Дополнительные сведения см. в разделе <a href="https://learn.microsoft.com/ru-ru/cpp/extensions/new-new-slot-in-vtable-cpp-component-extensions?view=msvc-170" data-linktype="relative-path"><code>new</code> (новый слот в vtable)</a></p>
</div>
<p>Если <strong><code>new</code></strong> используется для выделения памяти для объекта класса C++, конструктор объекта вызывается после выделения памяти.</p>
<p><a href="https://learn.microsoft.com/ru-ru/cpp/cpp/delete-operator-cpp?view=msvc-170" data-linktype="relative-path"><code>delete</code></a> Используйте оператор, чтобы освободить память, выделенную оператором<strong><code>new</code></strong>. <strong><code>delete[]</code></strong> Используйте оператор для удаления массива, выделенного оператором<strong><code>new</code></strong>.</p>
<p>В следующем примере выделяется и затем освобождается двумерный массив символов размером <code>dim</code> на 10. При выделении многомерного массива все измерения, кроме первого, должны быть константными выражениями, которые оцениваются положительными значениями. Самое левое измерение массива может быть любым выражением, которое оценивается положительным значением. При выделении массива <strong><code>new</code></strong> с помощью оператора первое измерение может быть равно нулю; <strong><code>new</code></strong> оператор возвращает уникальный указатель.</p>

<p>В следующем списке описываются элементы грамматики <strong><code>new</code></strong>:</p>
<p><em><code>new-placement</code></em><br>
Предоставляет способ передачи дополнительных аргументов при перегрузке <strong><code>new</code></strong>.</p>
<p><em><code>type-id</code></em><br>
Указывает тип, который нужно выделить; это может быть встроенный или определяемый пользователем тип. Если спецификация типа является сложной, она может быть окружена круглыми скобками, чтобы принудительно реализовать порядок привязки. Тип может быть заполнителем (<strong><code>auto</code></strong>), тип которого определяется компилятором.</p>
<p><em><code>new-initializer</code></em><br>
Предоставляет значение для инициализированного объекта. Инициализаторы не могут быть указаны для массивов. Оператор <strong><code>new</code></strong> создаст массивы объектов только в том случае, если класс имеет конструктор по умолчанию.</p>
<p><em><code>noptr-new-declarator</code></em><br>
Задает границы массива. При выделении многомерного массива все измерения, кроме первого, должны быть константными выражениями, которые оцениваются в положительные значения, преобразуемые <code>std::size_t</code>в . Самое левое измерение массива может быть любым выражением, которое оценивается положительным значением. Применяется <em><code>attribute-specifier-seq</code></em> к связанному типу массива.</p>

<div class="heading-wrapper" data-heading-level="h2"><a class="anchor-link docon docon-link" href="#initializing-objects-allocated-with-new" aria-label="Раздел с заголовком: Инициализация объектов, выделенных с помощью new"></a><h2 id="initializing-objects-allocated-with-new" class="heading-anchor">Инициализация объектов, выделенных с помощью <code>new</code></h2></div>

Дополнительные сведения о том, как выполняется инициализация, см. в разделе <a href="https://learn.microsoft.com/ru-ru/cpp/cpp/initializers?view=msvc-170" data-linktype="relative-path">"Инициализаторы</a>". В следующем примере показано, как использовать выражение инициализации с оператором <strong><code>new</code></strong> :</p>
<div class="codeHeader" id="code-try-5" data-bi-name="code-header"><span class="language">C++</span>

<p>В этом примере объект <code>CheckingAcct</code> выделяется с помощью <strong><code>new</code></strong> оператора, но инициализация по умолчанию не указана. Затем объект <code>SavingsAcct</code> выделяется таким же образом, за исключением того, что он явно инициализирован до 34,98. Так как 34.98 имеет тип <strong><code>double</code></strong>, конструктор, принимаюющий аргумент этого типа, вызывается для обработки инициализации. Наконец, тип <code>HowMuch</code> , отличный от класса, инициализирован до 43.0.</p>
<p>Если объект имеет тип класса и этот класс имеет конструкторы (как и в предыдущем примере), объект можно инициализировать оператором <strong><code>new</code></strong> , только если выполняется одно из этих условий:</p>
<ul>
<li><p>Аргументы, предоставленные в инициализаторе, соответствуют аргументам конструктора.</p>
</li>
<li><p>Класс имеет конструктор по умолчанию (конструктор, который можно вызвать без аргументов).</p>
</li>
</ul>
<p>Явное инициализация для каждого элемента не может выполняться при выделении массивов с помощью <strong><code>new</code></strong> оператора; вызывается только конструктор по умолчанию, если он присутствует. Дополнительные сведения см. в разделе <a href="https://learn.microsoft.com/ru-ru/cpp/cpp/default-arguments?view=msvc-170" data-linktype="relative-path">"Аргументы</a> по умолчанию".</p>
<p>Если выделение памяти завершается сбоем (<strong><code>operator new</code></strong> возвращает значение 0), инициализация не выполняется. Это поведение защищает от попыток инициализации данных, которые не существуют.</p>
<p>Как и при вызовах функций, порядок вычисления инициализированных выражений не определен. Кроме того, вы не должны полагаться на эти выражения, которые оцениваются полностью перед выделением памяти. Если выделение памяти завершается ошибкой, а <strong><code>new</code></strong> оператор возвращает ноль, некоторые выражения в инициализаторе могут не оцениваться полностью.</p>

<div class="heading-wrapper" data-heading-level="h2"><a class="anchor-link docon docon-link" href="#how-new-works" aria-label="Раздел с заголовком: Как работает new"></a><h2 id="how-new-works" class="heading-anchor">Как работает <code>new</code></h2></div>
<p>Выражение <em><code>new-expression</code></em> , <strong><code>new</code></strong> содержащее оператор, выполняет три действия:</p>
<ul>
<li><p>Находит и резервирует хранилище для объекта или объектов, которым нужно выделить память. По завершении этого этапа выделяется правильный объем хранилища, но он еще не является объектом.</p>
</li>
<li><p>Инициализирует объекты. После завершения инициализации имеется достаточно информации, чтобы выделенная память являлась объектом.</p>
</li>
<li><p>Возвращает указатель на объекты типа указателя, производные от <em><code>new-type-id</code></em> или <em><code>type-id</code></em>. Программа использует этот указатель для доступа к новому объекту, которому выделена память.</p>
</li>
</ul>
<p>Оператор <strong><code>new</code></strong> вызывает функцию <strong><code>operator new</code></strong>. Для массивов любого типа и для объектов, которые не <strong><code>class</code></strong>являются , <strong><code>struct</code></strong>или <strong><code>union</code></strong> типами, глобальной функцией, <strong><code>::operator new</code></strong>вызывается для выделения хранилища. Объекты типа класса могут определять собственную <strong><code>operator new</code></strong> статическую функцию-член на основе каждого класса.</p>
<p>Когда компилятор обнаруживает <strong><code>new</code></strong> оператор для выделения объекта типа<code>T</code>, он выдает вызов <code>T::operator new( sizeof(T) )</code> или, если пользователь не определен<strong><code>operator new</code></strong>. <code>::operator new( sizeof(T) )</code> Это то, как <strong><code>new</code></strong> оператор может выделить правильный объем памяти для объекта.</p>
<div class="alert is-info">
<p class="alert-title"><span class="docon docon-status-error-outline" aria-hidden="true"></span> Примечание</p>
<p>Аргумент <strong><code>operator new</code></strong> типа <code>std::size_t</code>. Этот тип определяется в &lt;direct.h&gt;, &lt;malloc.h, &lt;memory.h&gt;, search.h&gt;&gt;&lt;, &lt;stddef.h&gt;, &lt;stdio.h&gt;, stdlib.h&gt;, &lt;&lt;string.h&gt; и &lt;time.h.&gt;</p>
</div>
<p>Параметр в грамматике разрешает спецификацию <em><code>new-placement</code></em> (см. грамматику оператора<a href="https://learn.microsoft.com/ru-ru/cpp/cpp/new-operator-cpp?view=msvc-170" data-linktype="relative-path"><code>new</code></a>). Параметр <em><code>new-placement</code></em> можно использовать только для определяемых пользователем реализаций <strong><code>operator new</code></strong>; он позволяет передавать <strong><code>operator new</code></strong>дополнительные сведения. Выражение с таким полем <em><code>new-placement</code></em> , как <code>T *TObject = new ( 0x0040 ) T;</code> преобразование <code>T *TObject = T::operator new( sizeof( T ), 0x0040 );</code> в значение, в которое входит <code>operator new</code>класс T, в противном случае — в <code>T *TObject = ::operator new( sizeof( T ), 0x0040 );</code>.</p>
<p>Первоначальное <em><code>new-placement</code></em> намерение поля заключается в том, чтобы разрешить аппаратным зависимым объектам выделяться по указанным пользователем адресам.</p>
<div class="alert is-info">
<p class="alert-title"><span class="docon docon-status-error-outline" aria-hidden="true"></span> Примечание</p>
<p>Хотя в предыдущем примере показано только один аргумент в <em><code>new-placement</code></em> поле, нет ограничений на то, сколько дополнительных аргументов можно передать таким <strong><code>operator new</code></strong> образом.</p>
</div>
<p>Даже если <strong><code>operator new</code></strong> для типа <code>T</code>класса определен тип класса, можно явно использовать глобальный оператор <strong><code>new</code></strong> , как в следующем примере:</p>
<div class="codeHeader" id="code-try-7" data-bi-name="code-header"><span class="language">C++</span>
